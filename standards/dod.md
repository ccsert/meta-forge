# Definition of Done（完成定义）

> 一个任务什么时候算"完成"？以下所有条件必须满足。

---

## 代码层面

- [ ] 代码已按规范编写（参考 [coding.md](./coding.md)）
- [ ] 代码已通过 lint 检查，无 error
- [ ] 代码已通过类型检查（如适用）
- [ ] 没有遗留的 TODO / FIXME（或已记录为新 task）
- [ ] 没有 console.log / 调试代码残留

## 测试层面

- [ ] 核心逻辑有单元测试覆盖
- [ ] 关键路径有集成测试覆盖
- [ ] 所有测试通过

## 文档层面

- [ ] 相关文档已更新（README / API 文档 / 注释）
- [ ] 如有架构决策，已记录 ADR

## 任务层面

- [ ] 任务卡验收标准已全部满足
- [ ] 该任务在单 workspace 中完成并完成自验证
- [ ] 任务依赖的输入文档已被使用或更新
- [ ] VibeKanban 任务状态已更新为 done
- [ ] 依赖此任务的下游任务已被通知/解锁

## 质量层面

- [ ] 代码已自我 review 一遍
- [ ] 没有引入新的安全漏洞
- [ ] 性能没有明显退化

---

## 特殊情况

> 如果某条 DoD 无法满足，必须在任务卡中注明原因，并创建跟踪 task。
