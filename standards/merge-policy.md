# 合并策略

## 策略类型

### `auto-merge`
适用于无冲突且测试通过的改动。

### `agent-resolve`
适用于轻量、低风险、可验证的冲突。

### `escalate-issue`
适用于高风险或语义冲突场景。

## 必须升级为新 issue 的场景

- 数据库迁移冲突
- 公共接口契约冲突
- 安全策略冲突
- 架构边界调整冲突
- 同一核心文件的多任务语义冲突

## agent 可自行解决的场景

- README / 文档冲突
- 非核心模块的机械性文本冲突
- 格式化导致的轻量差异
- 明确可通过测试验证的局部代码冲突

## 决策要求

发生冲突时必须输出：

- 冲突分类
- 影响范围
- 处理策略
- 验证结果
- 是否创建 follow-up issue
