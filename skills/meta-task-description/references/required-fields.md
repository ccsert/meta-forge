# Required Fields Reference

Executable issue descriptions should explicitly include:

- Task Identity
- Execution Classification
- Relationships
- Repo Context
- Document Inputs
- Goal
- Acceptance Criteria
- Execution Notes

Container nodes like epic/feature/story can use the same structure, but should clearly mark:

- `Execution Mode: non-executable`
- `Workspace Launchable: false`
- a container-style execution reason
